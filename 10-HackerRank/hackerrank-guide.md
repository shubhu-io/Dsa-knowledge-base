## How Problems Work

### Problem Structure
- **Problem Statement**: Clear description of the task
- **Input Format**: Expected input structure
- **Output Format**: Expected output structure
- **Constraints**: Limits on input size and values
- **Sample Input/Output**: Examples to understand the problem
- **Explanation**: Why the output is correct

### Difficulty Levels
- **Easy**: Basic concepts, 10-20 minutes
- **Medium**: Intermediate skills, 20-40 minutes
- **Hard**: Advanced techniques, 40-90 minutes

### Scoring System
- Problems have point values based on difficulty
- Points are awarded for correct solutions
- Time bonus for faster solutions
- Partial scoring for some problems

## Interview Preparation

### Assessment Format

**Typical HackerRank Assessment**
- Duration: 60-120 minutes
- Problems: 2-5 coding questions
- Difficulty: Easy to Hard
- Languages: Multiple choice
- Environment: Online IDE

**Common Assessment Types**
- **Pre-interview screening**: 2-3 problems, 60 minutes
- **Technical interview**: 1-2 problems, 45-60 minutes
- **Coding challenge**: 3-5 problems, 90-120 minutes
- **Hiring contest**: Multiple problems, 2-3 hours

### Time Management During Assessment

**60-Minute Assessment**
```
0-5 min: Read all problems
5-15 min: Solve easiest problem
15-35 min: Solve medium problem
35-55 min: Solve hard problem (or optimize)
55-60 min: Review and submit
```

**90-Minute Assessment**
```
0-5 min: Read all problems
5-20 min: Solve 2 easy problems
20-50 min: Solve 1-2 medium problems
50-80 min: Solve hard problem
80-90 min: Review and submit
```

### Common Assessment Patterns

**Array Problems**
- Two Sum and variants
- Subarray problems
- Array manipulation
- Sliding window

**String Problems**
- Palindrome checks
- Anagram detection
- Substring search
- String transformation

**Linked List Problems**
- Reversal
- Cycle detection
- Merge operations
- Remove elements

**Tree Problems**
- Traversal (inorder, preorder, postorder)
- BST operations
- Tree height
- Path finding

**Graph Problems**
- BFS/DFS traversal
- Shortest path
- Connected components
- Cycle detection

**DP Problems**
- Fibonacci variants
- Knapsack problems
- Longest subsequence
- Grid paths

## Common Mistakes

### Technical Mistakes
1. **Off-by-one errors**: Check loop boundaries
2. **Integer overflow**: Use long for large numbers
3. **Null pointer exceptions**: Check for None/null
4. **Infinite loops**: Ensure termination
5. **Wrong data type**: Match problem requirements

### Strategy Mistakes
1. **Not reading all problems**: Always scan first
2. **Spending too long on one problem**: Move on if stuck
3. **Ignoring edge cases**: Test thoroughly
4. **Not optimizing**: Check time complexity
5. **Rushing submissions**: Take time to verify

### Practice Mistakes
1. **Skipping easy problems**: Build strong foundation
2. **Only reading solutions**: Active solving is key
3. **Ignoring weak areas**: Focus on improvement
4. **Not tracking progress**: Monitor improvement
5. **Inconsistent practice**: Daily practice matters
