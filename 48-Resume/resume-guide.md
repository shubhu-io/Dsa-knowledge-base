## Resume Structure

### Standard Layout

```
┌─────────────────────────────────────────────────────────┐
│                    CONTACT INFORMATION                   │
│  Name | Email | Phone | LinkedIn | GitHub | Location     │
├─────────────────────────────────────────────────────────┤
│                   PROFESSIONAL SUMMARY                   │
│  2-3 sentences highlighting key qualifications           │
├─────────────────────────────────────────────────────────┤
│                     WORK EXPERIENCE                      │
│  Most recent 2-3 positions with bullet points            │
├─────────────────────────────────────────────────────────┤
│                        SKILLS                            │
│  Technical skills, tools, and technologies               │
├─────────────────────────────────────────────────────────┤
│                       EDUCATION                          │
│  Degree, university, graduation date                     │
├─────────────────────────────────────────────────────────┤
│                  PROJECTS (Optional)                     │
│  Notable technical projects with outcomes                │
├─────────────────────────────────────────────────────────┤
│                   CERTIFICATIONS                         │
│  Relevant professional certifications                    │
└─────────────────────────────────────────────────────────┘
```

### Section Order by Experience Level

```markdown
**Entry Level (0-2 years)**:
1. Contact Information
2. Education (first)
3. Projects
4. Skills
5. Internships/Work Experience

**Mid Level (3-7 years)**:
1. Contact Information
2. Professional Summary
3. Work Experience
4. Skills
5. Education
6. Projects

**Senior Level (8+ years)**:
1. Contact Information
2. Professional Summary
3. Work Experience
4. Leadership/Impact
5. Skills
6. Education
7. Certifications
```

## Contact Information

### Essential Elements

```markdown
**Required**:
- Full Name (prominent)
- Professional Email
- Phone Number
- Location (City, State)

**Recommended**:
- LinkedIn Profile URL
- GitHub/Portfolio URL
- Personal Website

**Optional**:
- Professional Headshot (varies by country)
- Twitter (if tech-focused)
```

### Example Formats

```markdown
**Standard Format**:
JOHN DOE
john.doe@email.com | (555) 123-4567 | San Francisco, CA
linkedin.com/in/johndoe | github.com/johndoe

**Compact Format**:
John Doe | john.doe@email.com | (555) 123-4567 | SF, CA
LinkedIn: /in/johndoe | GitHub: /johndoe | Portfolio: johndoe.dev
```

```markdown
✅ Professional emails:

- john.doe.dev@proton.me

❌ Unprofessional emails:
- cooldude99@yahoo.com
- partygirl@hotmail.com
- john.doe@random-domain.com
```

## Professional Summary

### Formula for Strong Summary

```
[Title/Role] with [X years] of experience in [Key Skills].
Proven track record of [Major Achievement]. Expert in [Technical Skills]
with focus on [Specialization].
```

### Examples by Role

```markdown
**Software Engineer**:
"Results-driven Software Engineer with 5+ years of experience building
scalable web applications. Expertise in React, Node.js, and cloud
architectures. Led migration to microservices, improving system
performance by 40%. Passionate about clean code and user experience."

**Senior Developer**:
"Senior Software Engineer with 8+ years designing distributed systems
handling 10M+ daily users. Specialized in Python, AWS, and system
design. Architected real-time data pipeline processing 1TB+ daily.
Mentored team of 5 junior developers."

**DevOps Engineer**:
"DevOps Engineer with 4+ years automating CI/CD pipelines and
infrastructure. Reduced deployment time from 2 hours to 15 minutes
using Kubernetes and Terraform. Managed 99.99% uptime across
production environments."

**Full Stack Developer**:
"Full Stack Developer with 6+ years building end-to-end web solutions.
Proficient in React, Node.js, PostgreSQL, and AWS. Delivered 20+
features quarterly while maintaining 95% test coverage. Led agile
team of 4 developers."
```

### Common Mistakes

```markdown
❌ Too generic:
"Hardworking developer looking for new opportunities"

✅ Specific and results-oriented:
"Software Engineer with 5 years experience building scalable APIs
handling 1M+ requests daily using Node.js and AWS"
```

## Work Experience

### Bullet Point Formula

```
Action Verb + Task + Result/Impact
```

### Strong Action Verbs

```markdown
**Leadership**: Led, Directed, Managed, Coordinated, Mentored
**Development**: Built, Designed, Developed, Implemented, Created
**Improvement**: Optimized, Enhanced, Streamlined, Modernized
**Achievement**: Achieved, Exceeded, Delivered, Launched, Shipped
**Analysis**: Analyzed, Evaluated, Assessed, Researched, Identified
```

### Bullet Point Examples

```markdown
**Weak**:
- Worked on frontend development
- Responsible for API development
- Helped with database management
- Participated in code reviews

**Strong**:
- Developed React component library used by 50+ developers,
  reducing UI development time by 35%
- Designed and implemented RESTful API serving 10M+ daily
  requests with 99.9% uptime
- Optimized PostgreSQL queries reducing average response time
  from 500ms to 100ms
- Led code review process for team of 8, establishing standards
  that reduced bugs by 25%

**With Numbers**:
- Increased test coverage from 60% to 92% across 200+ components
- Reduced AWS costs by 30% ($15K/month) through infrastructure
  optimization
- Shipped 15 features in Q3, exceeding team target by 50%
- Managed migration of 500K users with zero downtime
- Built automated deployment pipeline reducing release cycle
  from 2 weeks to 1 day
```

### Experience Format

```markdown
**Senior Software Engineer** | TechCorp Inc. | Jan 2021 - Present
San Francisco, CA

• Architected microservices infrastructure handling 50M+
  monthly active users, improving scalability by 300%
• Led team of 6 engineers in rebuilding core platform,
  delivering 3 months ahead of schedule

  from 2 hours to 15 minutes
• Mentored 4 junior developers, with 2 receiving promotions

**Software Engineer** | StartupXYZ | Jun 2018 - Dec 2020
Remote

• Built real-time collaboration features using WebSockets
  serving 10K concurrent users
• Designed database schema supporting 100M+ records with
  sub-100ms query times
• Created automated testing framework increasing coverage
  from 45% to 88%

  optimization
```

## Formatting Guidelines

### Font Choices

```markdown
**Professional Fonts**:
- Calibri (11pt) - Modern, clean
- Arial (10-11pt) - Safe, readable
- Garamond (11-12pt) - Elegant, traditional
- Helvetica (10-11pt) - Clean, professional

**Avoid**:
- Comic Sans
- Papyrus
- Decorative fonts
- Fonts smaller than 10pt
```

### Layout Guidelines

```markdown
**Margins**: 0.5" to 1" on all sides
**Line Spacing**: 1.0 to 1.15
**Section Spacing**: 6-12pt between sections
**Length**: 
- Entry level: 1 page
- Mid level: 1-2 pages
- Senior level: 2 pages max

**File Format**:
- PDF (preferred) - maintains formatting
- DOCX (acceptable) - for ATS compatibility
- Never send as image or unusual format
```

### File Naming

```markdown
✅ Good:
- John_Doe_Resume.pdf
- JohnDoe_SoftwareEngineer_Resume.pdf
- resume-john-doe-2024.pdf

❌ Bad:
- resume.pdf
- final_v3_updated_REAL.pdf
- Document1.pdf
```

## Tailoring for Different Roles

### Software Engineer

```markdown
**Focus on**:
- Technical skills and languages
- System design experience
- Code quality metrics
- Problem-solving examples

**Keywords**: Algorithms, data structures, design patterns,
microservices, API development, testing, code review
```

### Frontend Developer

```markdown
**Focus on**:
- UI/UX implementations
- Framework expertise
- Performance optimization
- Cross-browser compatibility

**Keywords**: React, Vue, Angular, responsive design,
accessibility, performance, animations, state management
```

### Backend Developer

```markdown
**Focus on**:
- API design and development
- Database optimization
- System architecture
- Scalability achievements

**Keywords**: REST APIs, microservices, databases,
caching, security, performance, scaling
```

### DevOps Engineer

```markdown
**Focus on**:
- Infrastructure automation
- CI/CD pipelines
- Cloud architecture
- Monitoring and reliability

**Keywords**: AWS, Azure, Docker, Kubernetes, Terraform,
Jenkins, monitoring, automation, infrastructure as code
```

### Data Engineer

```markdown
**Focus on**:
- Data pipeline design
- ETL processes
- Big data technologies
- Data quality

**Keywords**: Spark, Hadoop, SQL, Python, ETL,
data warehousing, streaming, data modeling
```

## ATS Optimization

### Keywords to Include

```markdown
**Match job description keywords**:
- Technical terms from requirements
- Tools and technologies mentioned
- Soft skills they emphasize
- Industry-specific terminology

**Common ATS keywords**:
- Software Development
- Agile/Scrum
- REST APIs
- CI/CD
- Version Control (Git)
- Testing/Quality Assurance
- Database Management
- Cloud Computing
```

### ATS-Friendly Format

```markdown
✅ DO:
- Use standard section headings
- Include keywords naturally
- Use simple formatting
- Submit as PDF or DOCX
- Use standard fonts

❌ DON'T:
- Use tables for content
- Include headers/footers
- Use special characters
- Submit as image
- Use creative formatting
```
