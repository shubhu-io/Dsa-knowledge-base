# Codeforces Contest Strategy

This document provides strategies and tips for performing well in Codeforces contests.

## Contest Format

Codeforces contests typically follow these formats:
- **Div. 2**: 5 problems (A, B, C, D, E) in 2 hours
- **Div. 3**: 6 problems (A-F) in 2 hours, easier difficulty
- **Div. 1**: 5 problems (A-E) in 2.5 hours, advanced difficulty
- **Educational Rounds**: 6 problems in 2 hours

## Pre-Contest Preparation

### Mental Preparation
- Get a good night's sleep before the contest
- Have a light meal and stay hydrated
- Avoid distractions during the contest
- Set realistic goals based on your current rating

### Technical Setup
- Test your environment beforehand
- Have your code templates ready (fast I/O, common data structures)
- Use a comfortable IDE or code editor
- Ensure stable internet connection

## During the Contest

### Problem Selection Strategy
1. **Read all problems first** - Spend 5-10 minutes understanding each problem
2. **Start with the easiest** - Usually problems A and B
3. **Estimate difficulty** - Look at problem constraints and tags
4. **Don't get stuck** - If a problem takes too long, move on

### Time Management
- **First 30 minutes**: Solve A and B, attempt C
- **Next 45 minutes**: Focus on C and D
- **Remaining time**: Attempt E or debug earlier solutions

### Coding Best Practices
- Write clean, readable code
- Use meaningful variable names
- Test with sample inputs before submitting
- Handle edge cases (empty inputs, single elements, large values)

## Problem-Solving Techniques

### For Implementation Problems (A, B)
- Focus on accuracy over speed
- Carefully parse the problem statement
- Implement step by step
- Test with given examples

### For Greedy Problems (C)
- Identify the greedy choice property
- Prove the greedy algorithm is correct
- Consider sorting as a preprocessing step

### For Graph Problems (D, E)
- Identify the graph type (directed, undirected, weighted)
- Choose the right algorithm (DFS, BFS, Dijkstra, etc.)
- Pay attention to constraints (can affect algorithm choice)

### For Data Structure Problems
- Know your STL/container operations
- Consider time complexity of operations
- Use appropriate data structures (set, map, priority_queue)

## Common Pitfalls to Avoid

1. **Integer overflow** - Use `long long` in C++
2. **Off-by-one errors** - Be careful with array indexing
3. **Infinite loops** - Ensure loop termination conditions
4. **Memory limits** - Don't allocate too much memory
5. **Time limits** - Choose algorithms that fit within constraints

## Post-Contest Analysis

### Review Your Solutions
- Check if there's a more optimal approach
- Look at other contestants' solutions
- Understand the editorial solutions
- Learn from mistakes

### Virtual Contests
- Participate in past contests as virtual contests
- Simulate real contest conditions
- Analyze your performance

## Rating Progression Strategy

### For Beginners (Rating < 1200)
- Focus on problems A and B
- Practice basic algorithms (sorting, searching)
- Learn to read problem statements quickly

### For Intermediate (Rating 1200-1600)
- Start attempting problem C consistently
- Learn dynamic programming and graph algorithms
- Practice common problem patterns

### For Advanced (Rating 1600+)
- Focus on problems D and E
- Master advanced algorithms (segment trees, flows, etc.)
- Participate in Div. 1 contests
