# Getting Started Guide

A step-by-step guide to navigating this CS & Software Engineering Knowledge Base.

---

## Step 1: Choose Your Starting Point

Pick the path that matches your current level. Don't skip ahead — build a foundation first.

### Complete Beginner (New to Programming)

```
00-Getting-Started  (you are here)
  -> 01-Programming-Languages   (pick Python or JavaScript)
  -> 02-Problem-Solving          (learn to think like a programmer)
  -> 03-Time-Complexity          (Big O basics)
  -> 04-Data-Structures          (arrays, linked lists, hash maps)
  -> 05-Algorithms               (sorting, searching, recursion)
```

### CS Student (Know the Basics)

```
04-Data-Structures               (review and go deeper)
  -> 05-Algorithms               (DP, greedy, graph algorithms)
  -> 15-SQL                      (queries, normalization)
  -> 16-Computer-Networks         (TCP/IP, HTTP, DNS)
  -> 17-Operating-System          (processes, memory, scheduling)
  -> 11-System-Design             (scalability, distributed systems)
```

### Job Seeker (Preparing for Interviews)

```
02-Problem-Solving                (patterns: sliding window, two pointers, etc.)
  -> 03-Time-Complexity           (analyze your solutions)
  -> 04-Data-Structures           (master the core structures)
  -> 05-Algorithms                (interview-critical algorithms)
  -> 11-System-Design             (design interviews)
  -> 12-Low-Level-Design          (OOD, UML, class design)
  -> 48-Resume                    (build a strong resume)
  -> 49-Behavioral-Interviews     (STAR method, common questions)
```

### Experienced Developer (Upskilling)

```
11-System-Design                  (distributed systems, microservices)
  -> 30-DevOps                    (CI/CD, monitoring, SRE)
  -> 20-Docker                    (containerization)
  -> 24-Kubernetes                (orchestration)
  -> 27-AWS                       (cloud platform)
  -> 35-Cybersecurity             (security fundamentals)
  -> 38-AI                        (machine learning basics)
```

## Step 2: How Each Topic Folder is Structured

Every folder follows a consistent format:

```
XX-Topic-Name/
├── README.md                   # Overview, prerequisites, learning objectives
├── topic-main-guide.md         # Core concepts with explanations and examples
├── topic-code-examples.md      # Code in C, C++, Java, Python, JavaScript
├── topic-practice-problems.md  # Graded problems (Easy → Expert)
└── topic-interview-questions.md # Common interview questions with answers
```

**How to work through a folder**:
1. Read `README.md` for the overview
2. Study the main guide carefully
3. Run every code example yourself
4. Solve the practice problems in order
5. Test yourself with the interview questions

## Step 3: Effective Study Techniques

### Daily Practice Routine (30-60 minutes)

| Activity | Time | What to Do |
|----------|------|------------|
| Review yesterday's topic | 5 min | Recall key concepts without looking |
| New content | 20 min | Read one section, run code examples |
| Practice problems | 20 min | Solve 1-2 problems from the current topic |
| Quick review | 5 min | Summarize what you learned today |

### The Feynman Technique

1. Pick a concept you're studying
2. Explain it in simple terms (as if teaching someone else)
3. Identify gaps in your explanation
4. Go back to the material and fill those gaps
5. Repeat until you can explain it clearly

### Spaced Repetition Schedule

| Review | Timing |
|--------|--------|
| First review | 1 day after learning |
| Second review | 3 days later |
| Third review | 1 week later |
| Fourth review | 2 weeks later |
| Fifth review | 1 month later |

## Step 4: Build Projects Alongside Learning

Don't wait until you've "learned enough." Start building from week 2.

| After Learning | Build This |
|----------------|-----------|
| Programming basics | Calculator, to-do list, quiz app |
| Data structures | Cache (hash map), task scheduler (heap), autocomplete (trie) |
| Algorithms | Path finder (graphs), scheduler (greedy), image compressor (DP) |
| SQL | Design and query a database for a real scenario |
| System design | Design a URL shortener, chat app, or news feed |
| Docker + AWS | Deploy a full-stack app to the cloud |

---

## Step 5: Track Your Progress

Use a simple checklist. Mark each folder complete when you've:
- [ ] Read the README and main guide
- [ ] Run all code examples
- [ ] Completed at least 50% of practice problems
- [ ] Answered the interview questions from memory
