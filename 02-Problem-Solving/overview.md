# Problem Solving Overview

## What is Problem Solving?
The process of finding solutions to difficult or complex issues.

## The Problem Solving Process

### 1. Understand the Problem
- Read carefully
- Identify inputs and outputs
- Determine constraints
- Ask clarifying questions
- Work through examples

### 2. Devise a Plan
- Look for patterns
- Consider similar problems
- Break into subproblems
- Choose an approach:
  * Brute force
  - Divide and conquer
  - Dynamic programming
  - Greedy
  - Backtracking
  - Mathematical formula

### 3. Carry Out the Plan
- Implement your solution
- Write clean, readable code
- Test as you go
- Handle edge cases

### 4. Review and Extend
- Check correctness
- Analyze efficiency
- Consider improvements
- Generalize if possible

## Problem Solving Strategies

### 1. Work Backwards
Start from the goal and work toward the start.

### 2. Look for Patterns
Find similarities to something
3. Simplify the Problem
Solve a simpler version first, then generalize.

### 4. Use Analogies
How have you solved similar problems?

### 5. Break into Smaller Parts
Divide complex problem into manageable pieces.

### 6. Make an Organized List
Systematically enumerate possibilities.

### 7. Guess and Check
Try potential solutions and verify.

### 8. Use Variables
Represent unknowns with symbols.

### 9. Draw a Diagram
Visualize the problem.

### 10. Work Backwards
Start from desired result and reverse steps.

## Common Problem Types

### Decision Problems
Yes/no answers (e.g., Is this number prime?)

### Optimization Problems
Find best solution (e.g., Shortest path)

### Counting Problems
How many ways? (e.g., Number of paths)

### Search Problems
Find specific item (e.g., Find x in array)

### Constructive Problems
Build something with properties (e.g., Create schedule)

## Essential Techniques

### 1. Sorting First
Many problems become easier after sorting.

### 2. Two Pointers
Useful for arrays/strings, especially sorted ones.

### 3. Sliding Window
Efficient for substring/subarray problems.

### 4. Invariants
Find what stays constant during process.

### 5. Monotonicity
Look for quantities that only increase/decrease.

### 6. Test Cases
Create comprehensive test sets:
- Empty input
- Single element
- All same elements
- Already sorted
- Reverse sorted
- Large inputs
- Boundary values

## Debugging Strategies

### 1. Print Statements
Show variable values at key points.

### 2. Debugger
Step through code execution.

### 3. Rubber Duck Explain
Explain code line by line to an inanimate object.

### 4. Binary Search Bug
Comment out halves to isolate issue.

### 5. Simplify Input
Test with minimal examples.

### 6. Check Assumptions
Verify each assumption explicitly.

## Thinking Aloud Heuristics
When stuck, ask yourself:
- What do I know?
- What do I need to find?
- Have I seen this before?
- Can I simplify?
- What's the worst case?
- What's the best case?
- Are there patterns?
- Can I try small examples?
- What if I change one thing?

## Common Mistakes

### 1. Solving the Wrong Problem
Make sure you understand requirements.

### 2. Overcomplicating
Start simple, then refine.

### 3. Ignoring Edge Cases
Test boundaries and special cases.

### 4. Premature Optimization
Get it working first, then optimize.

### 5. Not Planning
Jumping straight to coding without thinking.

### 6. Giving Up Too Soon
Persistence is key in problem solving.

## Practice Approaches

### 1. Start Small
Begin with easy problems to build confidence.

### 2. Topic-Based Practice
Focus on one concept at a time (arrays, strings, etc.)

### 3. Timed Practice
Simulate interview/competition conditions.

### 4. Review Solutions
After solving, read others' solutions.

### 5. Explain Your Solution
Teaching reinforces learning.

### 6. Revisit Problems
Solve again after a few days.
