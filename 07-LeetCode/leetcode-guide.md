## How to Solve Problems Effectively

### The 5-Step Approach

1. **Read & Understand** (2 minutes)
   - Read the problem twice
   - Identify input/output types
   - Note constraints and edge cases

2. **Plan Your Approach** (5 minutes)
   - Think of brute force first
   - Identify the optimal pattern
   - Discuss time/space complexity target

3. **Code** (15-20 minutes)
   - Write clean, readable code
   - Use meaningful variable names
   - Handle edge cases as you go

4. **Test** (5 minutes)
   - Trace through examples manually
   - Test edge cases: empty, single element, duplicates
   - Verify time/space complexity

5. **Optimize** (5 minutes)
   - Can you reduce time or space?
   - Is there a better pattern?
   - What if input size doubled?

## Common Mistakes to Avoid

1. **Skipping Easy Problems**: Don't rush to Hard; fundamentals matter
2. **Only Reading Solutions**: Active solving builds muscle memory
3. **Ignoring Time Complexity**: Always analyze Big O
4. **Not Testing Edge Cases**: Empty arrays, single elements, overflow
5. **Memorizing Without Understanding**: Patterns transfer, code specifics don't
6. **Solving Random Problems**: Follow a structured plan
7. **Not Reviewing**: Revisit problems you solved 2-4 weeks ago
