## Recommended Study Plan

```
Month 1: Foundations
├── Learn one language deeply (Python or JavaScript)
├── Basic data structures (arrays, linked lists, stacks, queues)
├── Solve 20 easy LeetCode problems
└── Build one simple project

Month 2: Core DSA
├── Trees, graphs, hash tables
├── Sorting and searching algorithms
├── Solve 30 medium LeetCode problems
└── Start learning a framework

Month 3: Advanced Topics
├── Dynamic programming basics
├── Graph algorithms (BFS, DFS, Dijkstra)
├── System design fundamentals
├── Build a portfolio project
└── Solve 20 more medium + 5 hard problems

Month 4: Interview Prep
├── Mock interviews (2x/week)
├── Behavioral interview prep
├── System design practice
├── Resume refinement
└── Apply to target companies
```

## Common Interview Questions

1. **How should I structure my interview preparation?** Follow a 3-4 month plan: fundamentals first, then focused practice, then mock interviews and application.

2. **Is a CS degree necessary?** No, but foundational knowledge is. Self-taught engineers can succeed by demonstrating skills through projects and contributions.

3. **How many LeetCode problems should I solve?** Quality over quantity. Solve 150-300 problems thoughtfully rather than rushing through 500.

4. **Should I focus on one language?** Yes for interviews. Be expert in one language; familiarity with others comes naturally over time.

5. **How do I stay current with technology?** Follow newsletters (TLDR, Bytes), watch Fireship/NeetCode, attend meetups, and build projects with new tools.
